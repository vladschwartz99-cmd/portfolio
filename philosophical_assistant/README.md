# Philosophy Assistant

Репозиторий содержит проект, посвященный разработке локальной RAG-системы
для поиска и генерации ответов по философским текстам.

В проекте реализован: 
* автоматический сбор корпуса философской литературы из 
Project Gutenberg и Stanford Encyclopedia of Philosophy (SEP);
* преобразование текстов в эмбеддинги с помощью мультиязычной модели 
multilingual-e5-base; 
* индексация текстов в FAISS;
* использование локальной языковой модели Ollama Qwen3 для генерации 
ответов на основе найденного контекста.
* локальный веб-интерфейс на Streamlit

---

### Мотивация к реализации
Проект реализован в образовательных целях для изучения RAG-систем, 
семантического поиска и локальных LLM. Реализация данного проекта именно 
на основе философских текстов вызвана исключительно моей любовью к философии.

##  Установка и запуск

1. Для реализации потребуется установка Ollama, доступная для скачивания по 
[этой ссылке](https://ollama.com/download)

2. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/vladschwartz99-cmd/portfolio.git
   ```
   
3. Перейдите в директорию проекта:
   ```bash
   cd portfolio/Philosopher_assistant
   ```
   
4. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
   
5. Загрузите модель Ollama:
   ```bash
   ollama pull qwen3:8b
   ``` 

6. Запустите проект:
      1. Только работа GAR-системы
      ```bash
      python app.py
      ```
      2. Полное воспроизведение проекта
      ```bash
      python load_data.py
      python prepare_data.py    # (эмбеддинги формируются ~ 1 час на RTX 4050)
      python app.py 
      ```
Примечание: если при выполнении скриптов модель ollama выдает ошибку соединения,
запустите соединение вручную:
```bash
ollama serve
```

## Основные моменты работы RAG-системы

RAG-система, за счет использования мультиязычной модели векторизации текстов,
принимает вопросы на множестве языков и формирует ответ на том языке, на 
котором был задан вопрос.

После получения вопроса RAG-система векторизирует его и ищет k-ближайших соседей 
к нему среди эмбеддингов фрагментов текстов, затем формируя из них контекст.

Модель Ollama получает вопрос и контекст и формирует на их основе ответ. Модель
использует только получаемый ей контекст, не использует собственные знания и 
не додумывает ответ. При недостатке контекста модель выводит ответ о том, что 
не может сформировать ответ на основе имеющегося контекста

Помимо самого ответа модели, RAG-система выводит также список источников, которые
формировали контекст (название статьи для SEP / имя автора и название произведения
для Project Gutenberg)
