# Anime Recommendation System

## О проекте

Репозиторий содержит проект, посвященный разработке рекомендательной системы для аниме 
на основе пользовательских оценок и характеристик тайтлов. В проекте реализовано 
сравнительное исследование четырех моделей, основанных на разных подходах к 
рекомендациям:

* Базовая система рекомендаций на основе популярности;
* Контентно-ориентированная система на основе признаков тайтлов;
* Система коллаборативной фильтрации на основе SVD;
* Гибридная система, объединяющая контентный и коллаборативный подходы.

Основная цель — изучить, насколько эффективно разные подходы могут предсказывать 
предпочтения пользователей на основе истории их оценок и характеристик контента.

## Структура данных

1. Данные об аниме содержат:
   * ID тайтла
   * название
   * жанры
   * тип тайтла (TV, Movie, OVA и др.)
   * количество эпизодов
   * средний рейтинг
   * количество пользователей, добавивших тайтл в список

2. Данные об оценках пользователей содержат:
   * ID пользователя
   * ID тайтла
   * пользовательскую оценку от 1 до 10 (-1 в случае просмотра без проставления оценки)

## Исследовательский анализ данных (EDA)

Были исследованы:
* распределение тайтлов по типам;
* распределение средних рейтингов тайтлов;
* распределение пользовательских оценок;
* активность пользователей;
* разреженность матрицы «пользователь-тайтл»;


Анализ показал, что:
* пользователи значительно чаще ставят высокие оценки;
* большая часть взаимодействий приходится на небольшое число популярных тайтлов;
* матрица взаимодействий является крайне разреженной (99.14%).

## Разбиение данных на train и test

Для разбиения данных использовался подход Leave-10-Out.

Для каждого пользователя:
1. Выбирались тайтлы с оценкой не ниже 7 баллов.
2. Случайным образом отбирались 10 тайтлов в тестовую выборку.
3. Остальные взаимодействия использовались для обучения моделей.

Пользователи, имеющие менее 15 релевантных оценок, исключались из оценки, благодаря чему
взаимодействия каждого пользователя присутствовали и в train и в test.

Такой подход позволяет оценить то, насколько модель может рекомендовать действительно
релевантные тайтлы.

## Реализованные модели

---

### Базовая система рекомендациЙ (Baseline)


Рекомендует наиболее популярные тайтлы по количеству просмотров, исключая уже 
просмотренные пользователем.

---

### Контентно-ориентированная система рекомендациЙ (ContentBased)

Контентно-ориентированная модель строится на основе характеристик тайтлов:
* жанры;
* тип тайтла.

Алгоритм:
1. Построение TF-IDF представления признаков. 
2. Расчет косинусного сходства между всеми тайтлами. 
3. Отбор тайтлов, оцененных пользователем не ниже 7 баллов. 
4. Расчет взвешенного сходства всех тайтлов с понравившимися тайтлами.
5. Фильтрация от уже просмотренных тайтлов.
6. Формирование рекомендаций.

---

### Рекомендательная система коллаборативной фильтрации (Collaborative)

Коллаборативная модель использует данные о пользовательских оценках, 
над которыми модель SVD из библиотеки Surprise выполняет матричную 
факторизацию матрицы "пользователь-тайтл" и прогнозирует ожидаемую оценку 
пользователя для каждого непросмотренного тайтла. Рекомендации формируются 
на основе наиболее высоких предсказанных оценок.

---

### Гибридная система рекомендациЙ (Hybrid)

Гибридная модель объединяет два подхода:
* контентно-ориентированный; 
* коллаборативный.

Для каждого тайтла вычисляется итоговая оценка:

$Hybrid Score = α × Content Score + (1 − α) × Collaborative Score$

Параметр $α$ подбирался экспериментально, путем подсчета метрик модели при разных 
его значениях на подвыборке из тестовых данных

Лучшее качество было достигнуто при $α$ = 0.4, что показывает, что оба подхода 
несут полезную информацию и взаимно дополняют друг друга.

## Выводы

1. Базовая система рекомендаций показала наилучшие метрики качества. Вероятно, это 
связано с тем, что значительная часть пользователей взаимодействует с одними и теми
же популярными тайтлами, поэтому рекомендации наиболее популярных объектов часто 
оказываются релевантными для большого числа пользователей.

2. Контентно-ориентированная система уступает другим подходам, поскольку использует 
ограниченный набор признаков объектов. 

3. Система коллаборативной фильтрации превосходит контентно-ориентированную систему, 
однако её эффективность могла быть ограничена экстремальной разреженностью матрицы
"пользователь-тайтл".

4. Гибридная система не смогла превзойти базовую, однако существенно улучшило 
качество рекомендаций относительно отдельных контентно-ориентированной и 
коллаборативной моделей.
